# FreeBlazorExtended.Agent

Windows Service that registers with the FreeBlazorExtended hub, sends periodic heartbeats with host telemetry (CPU, memory, disk, installed-service inventory, IIS app pools) and listens for hub-issued commands to start/stop/restart/uninstall Windows Services and recycle/start/stop IIS application pools.

## Status

This project is the agent reference implementation for Feature 105 (Agent Monitoring & Management). The hub side lives in `FreeBlazorExtended/Services/AgentMonitoringService.cs` and the dashboard UI is at `/showcase/feature105-agent-monitoring`.

## Windows-only

The agent is **Windows-only by design.** It relies on:

- `System.ServiceProcess.ServiceController` for Windows Service control
- `Microsoft.Web.Administration` (referenced from `C:\Windows\System32\inetsrv\Microsoft.Web.Administration.dll`) for IIS
- `sc.exe` for service install/uninstall
- DPAPI for at-rest token encryption (production hardening)

## Linux-port research targets

If a future Linux variant is needed, every Windows-specific path in the source code is annotated with a `Linux equivalents to research` comment block. The top three targets across the project are:

1. **systemd** – `systemctl start/stop/restart/disable <unit>` is the closest 1:1 service-control match.
2. **nginx** – `server { }` and `upstream { }` blocks replace IIS sites and app pools.
3. **Caddy / Traefik** – reverse-proxy with auto-managed TLS for hosts where automatic-cert is wanted.

## Install (Windows Server)

1. Publish a self-contained executable:
    ```
    dotnet publish -c Release -r win-x64 --self-contained -o C:\publish\FreeBlazorExtended.Agent
    ```
2. Copy `C:\publish\FreeBlazorExtended.Agent` to the target host (e.g. `C:\Program Files\FreeBlazorExtended.Agent`).
3. Edit `appsettings.json` -- set `Agent:HubUrl` and `Agent:RegistrationKey` (one-time key generated by the hub).
4. Install as a service:
    ```
    sc.exe create FreeBlazorExtendedAgent binPath= "\"C:\Program Files\FreeBlazorExtended.Agent\FreeBlazorExtended.Agent.exe\"" start= auto
    sc.exe description FreeBlazorExtendedAgent "FreeBlazorExtended Agent -- reports host telemetry to the hub."
    sc.exe start FreeBlazorExtendedAgent
    ```

The agent will redeem the registration key on first start, persist the long-lived API token to `agent.token` next to the executable, connect to the hub via SignalR, and begin sending heartbeats.

## Configuration (appsettings.json)

| Key | Description |
|-----|-------------|
| `Agent:HubUrl` | Base URL of the hub (e.g. `https://hub.example.com`). |
| `Agent:RegistrationKey` | One-time key. Cleared after first successful registration. |
| `Agent:ApiClientToken` | Long-lived bearer token. Populated by the agent after registration. |
| `Agent:HeartbeatIntervalSeconds` | Heartbeat cadence. Default 30. |
| `Agent:AgentName` | Display name. Defaults to `Environment.MachineName`. |

## Hub commands handled

| Command | Action |
|---------|--------|
| `StartService` | `ServiceController.Start()` |
| `StopService` | `ServiceController.Stop()` |
| `RestartService` | Stop, then start |
| `UninstallService` | `sc.exe delete <name>` (after best-effort stop) |
| `RecycleAppPool` | `Microsoft.Web.Administration.ApplicationPool.Recycle()` |
| `StartAppPool` | `ApplicationPool.Start()` |
| `StopAppPool` | `ApplicationPool.Stop()` |

The agent reports each command's result back to the hub via the `ReportCommandResult` SignalR method.
