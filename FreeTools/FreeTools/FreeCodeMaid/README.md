# FreeCodeMaid

A Roslyn-based **C# member reorganizer** for the FreeTools suite. It fills the one gap that
`.editorconfig` + `dotnet format` cannot: **reordering the members of a type** (CodeMaid's signature
feature) — while **preserving each member's comment / XML-doc block** and following a **modular,
customizable rule set**.

It is the deliberate counterpart to the per-language style references in GuidesV2 (`055`–`059`).

## Design goal: change as little as possible

FreeCodeMaid is calibrated to a codebase's *existing* convention so that running it is nearly a no-op
— it only tidies genuine ordering slips, and leaves deliberately hand-ordered code alone. Three rules
make that work:

1. **It only sorts what the convention sorts.** By default that's **properties + indexers + methods**,
   merged into one **interleaved alphabetical list** (matching how FreeCRM orders them). **Fields,
   constructors, events, operators, enums and nested types keep their original order** — the author
   groups fields *by purpose* (services, then state), not alphabetically, so reordering them would be
   wrong.
2. **It preserves original spacing.** Members that don't move stay **byte-identical** — comments, XML
   docs and the author's blank-line grouping are untouched. (Whitespace is the formatter's job.)
3. **It skips by-purpose types automatically.** If sorting would move more than
   `MaxFractionReordered` (default **35%**) of a type's sortable members, the type is treated as
   deliberately ordered and left alone. This auto-skips controllers, nested DTOs, plugin examples,
   etc., without naming them.

Sorting uses, in order: **kind** (the `KindOrder` table) → optional **accessibility**
(`GroupByVisibility`, off) → optional **static-first** (`StaticMembersFirst`, off) → **alphabetical**
with a leading `_` ignored (so `_connectionString` sorts as `connectionString`).

## Safety (it would rather do nothing than do harm)

- **Skips types containing `#region` / `#if` / `#pragma`** — reordering across those scrambles them.
- **Skips vendored / generated / build output** (`Exclude`) and **by-purpose / generated files**
  (`SkipFiles`, e.g. `DataObjects*.cs`). Imported production code is never touched.
- **Skips machine-generated files** (`<auto-generated>` header).
- **Only rewrites a type when the member order actually changes.**
- **Aborts any file** where a post-rewrite check finds a member went missing or a syntax error was
  introduced — it re-parses its own output and compares the full member set before writing.

## Calibration result on FreeCRM

Run against the real FreeCRM repo with the shipped defaults, FreeCodeMaid changes **8 files
(~620 lines)** — every change a genuine alphabetical slip-fix in the `DataAccess` layer (which the
author keeps alphabetical). **33 by-purpose types are left untouched.** The result **builds with
0 errors** and is **idempotent** (a second run changes nothing).

## Usage

```
FreeCodeMaid <path> [options]
```

- `<path>` — a `.cs` file, a directory, or a `.sln` / `.slnx` / `.csproj` (defaults to the current dir).
- `--apply`, `--write` — rewrite files in place. Without it, runs as a **dry run**.
- `--check` — dry run (the default): reports what would change and exits `1` if anything would.
- `--config=<file>` — use a specific `freecodemaid.json`. Otherwise the tool searches upward from
  `<path>`, then falls back to built-in defaults.
- `--verbose`, `-v` — also list skipped/unchanged files.
- `--help`, `-h`.

Exit codes: `0` clean/applied · `1` changes needed (check mode — handy as a CI gate) · `2` error.

```powershell
dotnet run --project FreeCodeMaid -- C:\src\FreeCRM            # dry run (nothing written)
dotnet run --project FreeCodeMaid -- C:\src\FreeCRM --apply    # write changes
```

## Caveats

- **Field initializer order.** Fields are not sorted by default, so this is moot under the defaults;
  if you enable field sorting, beware initializers that read another field (`int b = a + 1;`) — that
  compiles but can change behavior.
- **`.razor` files are not processed.** FreeCodeMaid reorders `.cs` only; Razor `@code` order is left
  to the author (Razor pages are by-purpose — see GuidesV2 doc 056).

## Configuration

Drop a `freecodemaid.json` in the repo root (a fully-commented copy ships next to this README). Every
rule above is adjustable: the kind order and grouping, alphabetical vs. visibility grouping,
static-first, the by-purpose threshold, and the exclude / skip globs.
